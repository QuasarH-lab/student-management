任务总体与java视频教程中的学生管理系统演示较为接近，但项目开发都过程中使用了MVC模式，大量接口和HashMap，增加了一定的理解门槛。

## 一、MVC

> * MVC是 **Model-View-Controller** 的缩写，一种软件架构模式，核心思想是**将应用程序分为三个 interconnected 的组件**，以实现**关注点分离**，提高代码的可维护性、可扩展性和复用性。

------

> ### 三个核心组件

> ####  1. **Model（模型）**

> - **职责**：管理应用程序的**数据**、**业务逻辑**和**状态**。
> - **特点**：独立于用户界面，直接与数据库或数据源交互。
> - **示例**：用户数据、订单信息、数据验证规则。

> #### 2. **View（视图）**

> - **职责**：负责**数据的可视化呈现**和**用户交互界面**。
> - **特点**：只负责展示从 Model 获取的数据，不包含业务逻辑。
> - **示例**：HTML 页面、JSON 输出、移动端 UI 布局。

> #### 3. **Controller（控制器）**

> - **职责**：作为 **Model 和 View 之间的协调者**。
  > - 接收用户输入（如点击、请求）
  > - 调用 Model 处理业务逻辑
  - 选择 View 进行数据展示
> - **特点**：处理路由和请求分发，但不直接操作数据库。



### 二、HashMap

> Java 中的 `HashMap` 是 `java.util` 包中最常用的集合类之一，它实现了 `Map` 接口，以**键值对**的形式存储数据，允许使用 `null` 作为键和值，且不保证元素的顺序。下面介绍它的基本操作。
>
> ---
>
> ### 1. 创建 HashMap
> ```java
>  private final Map<String, Student> studentMap = new HashMap<>();
> 
>     @Override
>     public boolean add(Student student) {
>         // 若学号已存在，新增失败（返回false）；否则存入map（返回true）
>         String studentId = student.getStudentId();
>         if (studentMap.containsKey(studentId)) {
>             return false;
>         }
>         studentMap.put(studentId, student);
>         return true;
>     }
> ```
>
> ---
>
> ### 2. 添加 / 更新元素
> ```java
> map.put("apple", 5);      // 添加
> map.put("banana", 3);
> map.put("apple", 8);      // 键已存在，覆盖旧值，返回旧值 5
> ```
>
> ---
>
> ### 3. 获取元素
> ```java
> Integer value = map.get("apple");      // 返回 8
> Integer notExist = map.get("orange");  // 返回 null
> 
> // 获取默认值（Java 8+）
> Integer val = map.getOrDefault("orange", 0); // 返回 0
> ```
>
> ---
>
> ### 4. 删除元素
> ```java
> Integer removed = map.remove("banana"); // 返回被删除的值 3
> map.remove("apple", 8);                // 仅当键值对完全匹配时删除，返回 boolean
> map.clear();                           // 清空所有键值对
> ```
>
> ---
>
> ### 5. 检查是否包含
> ```java
> studentMap.containsKey(studentId)) 
> --------------------------------------------
> boolean hasValue = map.containsValue(8);    // 查询Value
> ```
>
> ---
>
> ### 6. 遍历 HashMap
> ```java
> // 方式1：遍历键（keySet）
> for (String key : map.keySet()) {
>     System.out.println(key + " = " + map.get(key));
> }
> 
> // 方式2：遍历值（values）
> for (Integer val : map.values()) {
>     System.out.println(val);
> }
> 
> // 方式3：遍历键值对（entrySet）—— 最常用
> for (Map.Entry<String, Integer> entry : map.entrySet()) {
>     System.out.println(entry.getKey() + " = " + entry.getValue());
> }
> 
> // Java 8+ 使用 Lambda
> map.forEach((k, v) -> System.out.println(k + " = " + v));
> ```
>
> ---
>
> ### 7. 其他常用操作
> ```java
> int size = map.size();        // 返回键值对数量
> boolean empty = map.isEmpty(); // 判断是否为空
> ```
>
> ---
>
> ### 8. 注意事项
> - **线程不安全**：多个线程同时读写 `HashMap` 会引发并发问题，建议使用 `ConcurrentHashMap` 或 `Collections.synchronizedMap()` 包装。
> - **遍历时修改**：在迭代过程中直接调用 `map.put()` 或 `map.remove()` 会抛出 `ConcurrentModificationException`，应使用迭代器的 `remove()` 方法。
>   ```java
>   Iterator<Map.Entry<String, Integer>> it = map.entrySet().iterator();
>   while (it.hasNext()) {
>       Map.Entry<String, Integer> entry = it.next();
>       if (entry.getValue() == 0) {
>           it.remove(); // 安全删除
>       }
>   }
>   ```
> - **null 键/值**：允许一个 `null` 键和多个 `null` 值。
>
> ---
>
> ### 底层结构简述
> `HashMap` 内部使用**数组 + 链表/红黑树**（Java 8+）存储元素。当链表长度超过阈值（默认为 8）且数组长度达到 64 时，链表会转为红黑树，提高查询效率。
>
> 掌握这些基本操作后，就可以熟练使用 `HashMap` 解决大部分键值存储的需求了。

### 三、基本思路

首先要理解各模块的基本功能

1.学生实体类：封装学生的基本信息
职责：仅作为数据载体，不包含业务逻辑
2.控制层：负责接收用户输入，调用Service层处理，展示结果
职责：不包含任何业务逻辑，仅做交互和调用转发
3.数据访问层实现类：用HashMap模拟数据库，实现Student Mapper接口的方法
职责：仅处理数据存储和读取，不做任何业务校验（校验交给Service层）
4.业务逻辑层实现类：实现具体业务逻辑，依赖Mapper层进行数据操作
职责：处理数据校验、业务规则，调用Mapper层方法完成数据操作

___其中 数据访问层实现类 与 业务逻辑层实现类 使用了接口来调用各种方法 同时也是任务重点___
__理解要求：__先读懂 学生实体类 与 控制层 的代码，明确任务需求
__创建方法：__ 确定各方法输入输出的数据类型，并按照格式命名
__填充各方法：__如果比较熟练可将 数据访问层实现类 与 业务逻辑层实现类 分层实现，本人对项目开发以及方法内部的实现不甚熟悉，只能先将其纵向完成

### 四、任务完成中的疑惑

__1.修改年龄时输入0存在异常__  Student 对象中age 的数据类型为 int ，默认值为0，当修改年龄为null时，age=0，无法通过 18-25 判断，只好将 =0作为条件加入，但此时输入0就不会提示，只会和输入为null一样在Student Mapper环节被排除

只能修改已完成的Student Controller，在输入时就将输入值为0的情形排除

__2.添加与修改时输入错误数据反馈不及时：__在输入错误数据时经常无法及时反馈，需要等到全部输入完成时才能得到失败信息，影响用户体验，但程序的分层结构使修改反馈较为复杂，可以在日后尝试修改。
